# Cases

Esta pasta guarda o registro factual de cases de sucesso da Find4You: um
arquivo por case, nunca dados inventados ou de exemplo tratados como reais.
Diferente de `docs/clientes/` (perfil contínuo do cliente), um case documenta
um processo específico já concluído, com resultado mensurável.

## Convenção

- Um arquivo por case: `docs/cases/case-<nome-do-cliente-kebab-case>.md`.
- Use `_template.md` como ponto de partida ao criar o case de um processo novo.
- Resultado, métricas e depoimento nunca são inventados — se o dado ainda não
  foi validado, o campo permanece `[TODO] Validar com Thiago`. Um case não
  deve ser usado em material público (site, proposta, post) enquanto tiver
  campos pendentes.
- Nunca referencie um case específico em `prompts/`, `design-system/` ou em
  material de marketing público sem que o cliente tenha autorizado a
  divulgação — mesmo caso interno, ver princípios de confidencialidade em
  `docs/processos/dados-e-privacidade.md`.

## Quando consultar

- `marketing-conteudo` e `website` (página `docs/website/pages/servicos.md`
  ou similar) ao buscar prova social real para uma peça — nunca criar case
  fictício para preencher uma peça de conteúdo.
- `propostas-comerciais`, se um case validado for relevante como referência
  para um cliente em prospecção.
