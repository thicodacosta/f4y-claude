# Clientes

Esta pasta guarda o contexto factual de cada cliente da Find4You: um arquivo
por cliente, nunca dados inventados ou de exemplo tratados como reais.

## Convenção

- Um arquivo por cliente: `docs/clientes/<nome-do-cliente-kebab-case>.md`.
- Use `_template.md` como ponto de partida ao criar o arquivo de um cliente novo.
- Nunca referencie um cliente específico em `prompts/`, `design-system/` ou em
  material de marketing público — isso viola o princípio de confidencialidade
  descrito em `docs/marca/tom-de-voz.md` e `docs/processos/dados-e-privacidade.md`.

## Quando consultar

As skills verticais (`recrutamento`, `executive-search`, `alocacao-tech`) devem
checar se já existe um arquivo do cliente aqui antes de um novo briefing, para
não repetir perguntas já respondidas em processos anteriores (ver
`docs/processos/fluxo-comercial.md`, seção "Intake").

## Conteúdo esperado por arquivo

Ver `_template.md` para a estrutura completa: sobre a empresa, mercado,
principais tecnologias, estrutura de TI, perfil dos profissionais contratados,
cultura, diferenciais, histórico da Find4You, lições aprendidas e observações.
Nenhum campo deve ser preenchido com informação não confirmada — onde não
houver dado documentado, manter `[TODO] Preencher com Thiago`.
